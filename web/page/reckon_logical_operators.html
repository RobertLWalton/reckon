Higher precedence operators are evaluated before lower precedence operators. For example, in x AND NOT y, the higher precedence NOT in the subexpression NOT y is evaluated first and then the lower precedence AND is evaluated.

Logical Expression Evaluation

Logical operators deal with the `truth values' `true' and `false'. These are special values that are NOT numbers or strings or objects. The true special value prints as *TRUE*, and a global variable is defined with the name *TRUE* whose value is the true special value. Similarly the false special value prints as *FALSE* and has a global variable of the same name whose value is the false special value.

Logical expressions are evaluated according to the operators in them:

IF ELSE (selectors)
The P-expression consists of an alternating sequence of selection operators and operands beginning and ending with an operand. The IF and ELSE operators alternate, with IF being first and ELSE being last. Thus there are an even number of operators and an odd number of operands.

The P-expression begins with a sequence of clauses of the form
value-operand IF condition-operand
that are separated by ELSE operators, with each such clause associating a value-operand with a condition-operand. This sequence if followed by a P-expression end clause of the form
ELSE  value-operand

Evaluation proceeds by evaluating condition-operands left to right in order to select a value-operand which is evaluated to produce the value of the P-expression. The first condition-operand to evaluate to *TRUE* selects its associated value-operand. If no condition expression evaluates to *TRUE*, the end clause value-operand is selected.

For example:

"less" IF x < y ELSE "more" IF x > y ELSE "equal"

AND
The expression consists of an alternating sequence of operators and operands beginning and ending with an operand.
