RECKON Functions

A function is a block of code that executes when the function is called. The call can pass values called arguments to the block and the block can return values called return values to the call. The code block can have side effects which change program data not referenced by the arguments.

A complete set of all the examples for this page is in reckon_functions.rec .

Function Declarations

Examples: 

function add (X) (Y):
    Z = X + Y
    return Z

Z = add (5) (7)
12

function (X) add (Y): return X + Y

next Z = (5) add (7)
12

function add (X, Y): return X + Y

next Z = add ( 5, 7 )
12

Functions are defined by function declarations that consist of the word `function' followed by a function prototype followed by a colon `:' followed by either a restricted statement or a block. The restricted statement or block is executed when the function is called. A return statement in a function block returns values to a call of the function. A function restricted statement may be a return statement that returns values to a call.

A call to the function typically looks like the function prototype, except that arguments in the prototype are variable names called argument names, while arguments in the call are expressions which are evaluated to produce argument values passed by the call to the restricted statement or block executed by the function. In the restricted statement or block an argument name can be used to access the associated argument value passed by the call.

The return statement consists of the word `return' followed by zero or more comma separated expressions. Each expression computes one value to be returned to the function's caller. If the caller requires fewer values to be returned, values are discarded from the end of the list of returned values.

Calls to a function must occur after the declaration of the function. A function F2 may be declared inside the code block of a function F1, in which case calls to F2 must be inside this code block and after the declaration of F2.

A function prototype is an sequence of term names and argument lists. A term name is a sequence of words, natural numbers, marks, and separators not beginning with a natural number. In the prototype, marks and separators must be " quoted, and words may optionally be " quoted. The term names of a prototype of a function must match term names in a call to the function exactly, except that " quotes are removed from the term names in the call.

A term name and the argument lists that follow it (before the next term name) are called a term. Each term in a call is matched to the term in the prototype that has the same term name. However, the terms in the call may be in a different order than the terms in the prototype, except that the first call term name must match the first prototype term name.

A function does not need to have any term names provided it has at least two argument lists, nor does it need to have any argument lists if it has one term name.

Default Values

A function prototype can supply a default value for an argument. Then if a call to the function does not supply a value for the argument, the default value will be used.

To provide a default value for a prototype argument replace in the prototype the argument-name by:

argument-name ?=  default-value

Here default-value is any expression, and ?= is an operator with precedence just a bit higher than that of the comma operator.

Default value expressions are evaluated in the context of the call (not, like C++, the context of the function prototytpe).

In a prorotype argument list an argument with a default value cannot be before an argument without any default value.

An argument list is called omittable if all its arguments have default values. An omittable argument list may not immediately precede:

  • the first prototype term name, or
  • a non-omittable argument list with the same kind of brackets (e.g., both are `()'), or
  • an omittable argument list with a different kind of brackets (.e.g, one is `()' and one is `[]').
